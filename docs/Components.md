# AZVA Component System

## Component Philosophy

Every component should feel necessary.

No decoration without function.

## Four Content Types

### Statement
Large editorial headline or principle.

### Narrative
Supporting explanation.

### Framework
Structured model, diagram, or system.

### Evidence
Metric, proof point, case study, or product insight.

## Core Components

- Navbar
- Footer
- Section
- Section Closing
- Button
- Editorial Row
- Framework Grid
- Capability List
- PRIZM Score Card
- Founder Block
- Contact CTA

## Interaction Rule

Components should not compete with content.

The best interface is quiet until needed.

## Button Rules

Primary buttons are used sparingly.

Text links and underlined CTAs should be preferred when a premium editorial tone is needed.

## Card Rules

Use cards only when grouping is meaningful.

Do not use cards as default decoration.
